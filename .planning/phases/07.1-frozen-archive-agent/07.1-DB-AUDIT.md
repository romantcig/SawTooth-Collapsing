# Phase 07.1 Archive 数据库 Dry-Run 审计

## 安全边界

- 审计时间：2026-07-11。
- 生产来源：`data/sawtooth.db` 及当时存在的 `-wal`、`-shm` 配套文件。
- SQLite **没有再次直接打开生产路径**。测试仅以普通只读文件句柄复制三件套到 `t.TempDir()`，随后以包含 `mode=ro` 的 DSN 打开临时快照副本。
- 审计代码不调用 `NewSQLiteStore`、`createSchema`、`SaveArchive`、迁移、删除、写 PRAGMA、`VACUUM`、`REINDEX` 或 WAL checkpoint。
- 本报告是 dry-run；历史重复行保持原样。任何真实删除或迁移都需要老大另行明确授权。

此前一次直接 `mode=ro` 安全门验证曾使生产 `data/sawtooth.db-shm` 的 mtime 发生变化，但文件大小和 SHA-256 未变。该事实没有被回滚或掩盖；恢复执行后改为快照副本审计，以下不可变性证据均以恢复开始时的文件状态为基线。
