## Phase Boundary

<decisions>
## Implementation Decisions

### 自动会话标题辅助请求

- **D-01:** 高置信识别自动会话标题后执行完全安全直通：跳过 Frozen、Archive 召回与写入、压缩、Collapse、Decay、Eager Stub 及其他 session 状态副作用；保留安全 debug、上游转发和响应 usage。
- **D-02:** 使用双层线级识别。强路径为单条 user、`<session>...</session>`、标题生成 system、title-only JSON Schema；当通道能力门闩使 `output_config` 缺失时，必须同时满足单条 user、`<session>` 和明确标题 system 才可命中。
- **D-03:** `tools` 是否为空、`thinking` 是否显式 disabled、model 是否为 Haiku/小模型都不是硬条件；这些只是 Claude Code 中间构造意图，第三方通道最终 wire 可以不同。
- **D-04:** 本 Phase 只识别自动会话标题，不扩展 rename、teleport、compact、prompt suggestion、memory extraction 等其他 auxiliary 请求。
- **D-05:** 旁路输出一条不含正文的 Info 审计日志，记录 `request_id`、消息数、`request_kind=session_title` 和受限分类原因；不得记录 `<session>` 正文、system 全文或凭证。

### Archive 召回信号

- **D-06:** 精确文件/目录路径不再作为独立 Archive 召回信号。项目路径在正常编码对话中太常见，不能仅因路径匹配就搜索或注入历史。
- **D-07:** Archive 召回只允许由明确恢复意图或已有 `deep_search(...)` 提示触发；普通路径、工具名和公共词不得单独触发。
- **D-08:** 移除精确路径独立触发后，标题请求和普通主请求都不应仅因出现 `C:\Users\...`、`/home/...` 或 `internal/proxy/file.go` 而产生 Archive 召回汇总或 token 成本。

### the agent's Discretion

- 标题 system 意图匹配的具体受限锚点、分类 reason 枚举和纯函数拆分方式，只要同时满足严格误判防护与测试可覆盖性。
- 删除 `RecallSignalExactPath` 还是保留类型但不从普通路径生成信号，由规划与实现决定；最终外部行为必须符合 D-06/D-07。

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets

- `agentRequestFeatures`/`agentClassification`：可复用“脱敏特征提取 + 受限 reason + 早返回”模式，但辅助请求应有独立 request kind，不能伪装成 subagent。
- `requestMeta.Logger` 与 raw/forwarded debug：可用于标题旁路审计和离线 fixture 因果对比，不需要新增含正文日志。
- `extractRecallSignals`：召回信号集中在纯函数入口，适合移除普通精确路径触发并补充回归测试。

### Established Patterns

- 可靠 subagent 已在 Frozen/Archive/压缩副作用前安全早返回；标题辅助请求应遵循同类控制流，但保留自身明确的分类角色与日志原因。
- 生产管线只有在解析成功后才结构化重建；无法确认身份或协议时偏向透明、安全行为，禁止猜测式跨 session 或协议降级。

### Integration Points

- 标题分类必须发生在任何 Sawtooth request sequence、Frozen、Archive、Decay 或压缩状态副作用之前；当前 `Sawtooth.IncrementRequestSeq` 位于 body 分类之前，规划需审计并调整旁路边界。
- `searchAndExpandWithMeta` 当前在 `extractRecallSignals` 看到普通精确路径后即记录 Attempted；D-06/D-07 要求从信号生成入口消除该行为。

</code_context>
