# Quick Task 260711-12v Plan

## 目标

在保留现有 `slog` 调用和结构化属性的前提下，用定制 Handler 输出 YesMem 风格的日志：`[proxy] YYYY/MM/DD HH:MM:SS 消息 key=value`。终端按 YesMem 色板着色，非终端输出不包含 ANSI 转义码。

## 任务

1. 添加 Handler 与单元测试
   - 文件：`internal/proxy/loghandler*.go`、`internal/proxy/loghandler_test.go`
   - 验证：短时间戳、级别色、业务语义色、非 TTY 去色、属性与 Handler 契约均有测试。

2. 接入入口并标记现有关键业务日志的 YesMem 语义色
   - 文件：`cmd/proxy/main.go`、`internal/proxy/proxy.go`
   - 验证：不改动日志消息和业务流程，只替换 Handler 并给对应日志追加内部颜色属性。

3. 全量验证
   - 验证：`gofmt`、`go test ./...`、`go vet ./...` 通过；检查日志输出文件路径不出现 ANSI 转义码。

## 完成标准

- 默认日志不再输出 `time=... level=... msg=...`。
- 时间戳与 YesMem `log.LstdFlags` 一致。
- Windows 终端支持 ANSI 时显示 YesMem 色板；重定向或文件输出为纯文本。
- 老大已有的未提交修改保持不变。
