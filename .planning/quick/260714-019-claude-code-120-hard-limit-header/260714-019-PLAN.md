**已确认的库语义（Context7，执行时无需重新猜测）：**
- Go `http.Client.Timeout` 包含连接、重定向和 response body 读取，`0` 表示无 client-level 总 timeout；因此它不能继续承担模型完整生命周期的 120 秒上限。
- `http.Transport.ResponseHeaderTimeout` 只计算请求（含 body）完全写出后等待响应头的时间，不限制 response body 读取。
- 入站 `Request.Context()` 在客户端连接关闭或请求取消时会取消；出站请求 context 控制 Transport 取消。继续以 `r.Context()` 为父 context。
- yaml.v3 可将 `15s`、`10m`、`30m` 等标量解析到 `time.Duration`，并可将 `0` 解析为零 duration。

**设计边界：**
- 不照搬 YesMem 的 `ResponseHeaderTimeout=60s`；该值不适合长 non-stream thinking。本任务只借鉴其显式 Transport、Connection 清理和较长总预算的意图。
- 不创建每请求全新的 Transport。Transport 持有连接池，必须长期复用。为实现 stream/non-stream 两个响应头预算，使用一个 routing RoundTripper 持有两套长期复用的 `*http.Transport`，并由请求 context 中的私有 stream marker 选择；不在每次请求中 clone Transport。
- Go/HTTP 无法在 SSE 200 或 body 已提交后把状态改成 504。ST-owned timeout 在响应未提交时必须返回 504；已提交 SSE 时只能停止流并记录 `response_committed=true`。这不是错误分类降级，而是 HTTP wire 约束。
- 现有 `Server.HTTPClient *http.Client` 保留为测试注入 seam；默认构造改为 `Timeout: 0` 加自定义 Transport。
- 生产与测试代码注释使用简体中文。命令默认 Bash；race 测试必须先加载 Windows MinGW-w64 工具链。
